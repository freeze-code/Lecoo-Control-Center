## ⚠️ Важное предупреждение

Это программное обеспечение напрямую взаимодействует с аппаратным обеспечением вашей системы, в частности с окном HRAM встроенного контроллера (ITE IT5570/IT8987) и низкоуровневыми портами ввода-вывода. Неправильная настройка (например, установка пользовательской кривой вентиляторов на 0 RPM при высокой нагрузке) может привести к перегреву и необратимому повреждению оборудования.

Используя это программное обеспечение, вы осознаете эти риски. Автор не несет ответственности за любой ущерб, причиненный вашему устройству. Используйте на свой страх и риск.

## Возможности

  * **Мониторинг системы:** Чтение температур процессора/системы и скорости вращения вентиляторов (RPM).
  * **Управление питанием:** Переключение между предустановленными профилями питания EC (Тихий, Стандартный, Производительный).
  * **Управление охлаждением:** Независимое управление вентиляторами CPU и GPU (Автоматически, Полная скорость или Пользовательские значения скважности ШИМ).
  * **Состояние батареи (FlexiCharger):** Установка лимитов заряда для продления срока службы батареи (Полный, Высокий, Сбалансированный, Долговечный, Настольный режим).
  * **Управление подсветкой:** Регулировка яркости подсветки клавиатуры.
  * **Управление задним светодиодным кольцом:** Настройка заднего индикатора питания (статичная яркость или аппаратная анимация «дыхания»).

Это ПО разрабатывается и тестируется в первую очередь на Lecoo Pro 14 (Lecoo N155). Поддержка различных ревизий отслеживается ниже:

| Модель | Ревизия мат. платы | EC-чип | Статус |
| :--- | :--- | :--- | :--- |
| Lecoo Pro 14 Amd (H255) | N155A | IT5571-07 | Подтверждено (работает) |
| Lecoo Pro 14 Intel (Core Ultra 5) | N155D | IT5570-02 | Работает, кроме управления яркостью кнопки питания |
| Lecoo Pro 14 Intel (i5-13420H) | N155C | IT5570? | Вероятно, работает |

**Примечание:** Теоретически это ПО может работать на других ноутбуках на базе шасси Emdoor, использующих контроллеры ITE IT5570 или IT8987, так как демон включает базовую функцию автоопределения смещения HRAM.

Если вы успешно запустили программу на не указанной в списке ревизии оборудования или на другом шасси Emdoor, пожалуйста, откройте Issue или свяжитесь со мной, чтобы обновить список совместимости!

## Установка

### ⚠️ Рекомендуется: Используйте готовые бинарники

**НЕ собирайте из исходников, если вы не разработчик!** Вместо этого скачайте последний готовый релиз:

👉 **[Скачать последний релиз](https://github.com/LaVashikk/Lecoo-Control-Center/releases/latest)**

#### Установка в Windows

1. Скачайте архив `lecoo-*-windows.zip` со страницы релизов.
2. Распакуйте архив в любую папку.
3. Кликните правой кнопкой мыши на `install.bat` и выберите **"Запуск от имени администратора"**.
4. Откройте новое окно терминала и выполните `lecoo-ctrl help` для проверки установки.

#### Установка в Linux

1. Скачайте архив `lecoo-*-linux.tar.gz` со страницы релизов.
2. Распакуйте архив: `tar -xzf lecoo-*-linux.tar.gz`
3. Перейдите в распакованную папку и выполните: `sudo ./install.sh`
4. Используйте команду `lecoo-ctrl` для взаимодействия с демоном.

## Известные проблемы

* **Автозапуск службы в Windows 11:** На данный момент фоновая служба не запускается автоматически при старте Windows 11. Причины выясняются.
* **Сброс FlexiCharger при длительном обесточивании:** Если ноутбук выключен и отключен от зарядного устройства более 5 минут, микроконтроллер (EC) очищает память, сбрасывая лимиты зарядки. Если подключить питание *до* включения ноутбука, батарея зарядится до 100%. Однако после загрузки системы и старта службы лимиты применятся снова, и батарея плавно разрядится до установленного вами значения.
* **Индикатор заряда при Custom LED:** При использовании пользовательских настроек заднего светодиодного кольца (`led custom`), штатный индикатор заряда батареи перестает функционировать.
* **Залипание подсветки после жесткого выключения (Hard-off):** Если жестко выключить ноутбук (долгим удержанием кнопки питания) в режиме `led custom`, заднее кольцо продолжит светиться. **Решение:** Включите ноутбук и сразу же выключите его штатным образом (через меню ОС).
* **Конфликт со штатным ПО:** Управление профилями питания (команда `power`) может конфликтовать с официальной утилитой от производителя (`PowerModeUtility`). Рекомендуется не использовать их одновременно.
* **Античиты (Windows):** Некоторые античиты (например, FaceIT) могут блокировать или завершать процесс демона. Это происходит из-за использования официального драйвера производителя для доступа к железу.
* **Secure Boot и Kernel Lockdown (Linux):** На дистрибутивах с включенным Secure Boot (например, Fedora) демон не может получить доступ к портам ввода-вывода (`/dev/port`) из-за политики безопасности ядра. Это будет исправлено в будущих версиях.

## Использование (CLI)

Демон работает в фоновом режиме. Вы взаимодействуете с ним с помощью утилиты командной строки `lecoo-ctrl`.

<img src="branding\cli.jpg" alt="lecoo-ctrl" width=50% >

Основные команды для `lecoo-ctrl`:

### Информация о системе и мониторинг

  * `lecoo-ctrl help` - Показать доступные команды и их использование.
  * `lecoo-ctrl info` - Получить базовую информацию об EC и версии демона.
  * `lecoo-ctrl temps` - Отобразить текущие температуры процессора и системы.
  * `lecoo-ctrl fans` - Отобразить текущую скорость вращения вентиляторов CPU и GPU в RPM.

### Настройки питания и батареи

  * `lecoo-ctrl power <silent|default|perf>` - Применить определенный профиль питания/TDP.
      * *Пример:* `lecoo-ctrl power perf`
  * `lecoo-ctrl charge <full|high|balanced|lifespan|desk>` - Установить пороги зарядки батареи (FlexiCharger).
      * *Пример:* `lecoo-ctrl charge desk` (Ограничивает зарядку до 40-50% для постоянной работы от сети).
      * Запустите `lecoo-ctrl charge` без аргументов, чтобы просмотреть текущий лимит и емкость батареи.

### Управление охлаждением

  * `lecoo-ctrl fan <cpu|gpu> <auto|full|custom> [value]` - Управление поведением вентиляторов.
      * *Пример (Автоматически):* `lecoo-ctrl fan cpu auto`
      * *Пример (Максимальная скорость):* `lecoo-ctrl fan gpu full`
      * *Пример (Пользовательский ШИМ):* `lecoo-ctrl fan cpu custom 150` (Устанавливает пользовательское значение ШИМ).

### Управление подсветкой

  * `lecoo-ctrl kbd <0|1|2|3>` - Установить уровень подсветки клавиатуры (0 — выключена, 3 — максимум).
  * `lecoo-ctrl led <auto|custom>` - Управление задним светодиодным кольцом.
      * *Пример:* `lecoo-ctrl led custom 50`

## Телеметрия и сбор данных

Чтобы помочь улучшить программное обеспечение — в частности, для доработки логики автоопределения HRAM на различных ревизиях материнских плат и отслеживания непредвиденных сбоев демона — в этот проект включена **опциональная, полностью анонимная система телеметрии**.

**Что собирается:**

  * ТОЛЬКО данные микроконтроллера (версия чипа EC, смещение памяти HRAM).
  * Название процессора (CPU).
  * Базовое рабочее состояние (температуры, RPM вентиляторов, активный профиль питания).
  * Журналы сбоев (трассировки panic) в случае падения демона.

**Что НЕ собирается:**

  * Абсолютно НИКАКИЕ данные операционной системы, имена пользователей, IP-адреса, MAC-адреса или личная информация.

Телеметрия включена по умолчанию для поддержки развития проекта. Если вы хотите отказаться, вы можете отключить её в любой момент с помощью следующей команды:

```bash
lecoo-ctrl daemon telemetry disable
````

## Сборка из исходников (для разработчиков)

Убедитесь, что у вас установлен набор инструментов (toolchain) Rust.

Клонируйте репозиторий:

```bash
git clone https://github.com/LaVashikk/Lecoo-Control-Center.git
cd Lecoo-Control-Center
```

Вы можете собрать проект, используя стандартные команды Cargo, или использовать заранее определенные алиасы, расположенные в `.cargo/config.toml`:

**Для Windows:**

```bash
cargo build-win       # Собирает демона
cargo build-ctrl-win  # Собирает CLI-клиент
```

**Для Linux:**

```bash
cargo build-linux       # Собирает демона
cargo build-ctrl-linux  # Собирает CLI-клиент
```
